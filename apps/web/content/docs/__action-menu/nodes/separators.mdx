## Overview

Separators are simple visual dividers used to organize menu sections. They create visual breaks between groups of related items, improving menu readability and organization.

## Basic Usage

```tsx
const menu = {
  nodes: [
    { kind: 'item', id: 'cut', label: 'Cut' },
    { kind: 'item', id: 'copy', label: 'Copy' },
    { kind: 'item', id: 'paste', label: 'Paste' },
    { kind: 'separator', id: 'sep-1' },
    { kind: 'item', id: 'undo', label: 'Undo' },
    { kind: 'item', id: 'redo', label: 'Redo' },
  ],
}
```

## Type Definition

```tsx
type SeparatorDef = {
  kind: 'separator'
  id: string
  label?: string
  hidden?: boolean
}
```

## Properties

### Required Properties

#### `kind`
- **Type**: `'separator'`
- **Description**: Must be `'separator'` to identify this node as a separator

#### `id`
- **Type**: `string`
- **Description**: Unique identifier for the separator within its sibling nodes

### Optional Properties

#### `label`
- **Type**: `string`
- **Description**: Optional label for the separator (can be used for section headers)
- **Default**: `undefined`

```tsx
const menu = {
  nodes: [
    { kind: 'item', id: 'bold', label: 'Bold' },
    { kind: 'item', id: 'italic', label: 'Italic' },
    { kind: 'separator', id: 'sep-1', label: 'Text Alignment' },
    { kind: 'item', id: 'left', label: 'Align Left' },
    { kind: 'item', id: 'center', label: 'Align Center' },
  ],
}
```

#### `hidden`
- **Type**: `boolean`
- **Description**: Controls whether the separator is visible
- **Default**: `false`

```tsx
const menu = {
  nodes: [
    { kind: 'item', id: 'item1', label: 'Item 1' },
    { kind: 'separator', id: 'sep-1', hidden: true }, // Won't be rendered
    { kind: 'item', id: 'item2', label: 'Item 2' },
  ],
}
```

## Default Rendering

By default, separators render as a simple `<div>` with the role `"separator"`:

```tsx
<div role="separator" data-slot="action-menu-separator" />
```

The styling is controlled by your theme's `classNames.separator` configuration.

## Customization

### Custom Separator Component

You can customize how separators are rendered using the `slots.Separator` theme property:

```tsx
const theme = {
  slots: {
    Separator: ({ node }) => (
      <div className="my-separator">
        {node.label && <span className="separator-label">{node.label}</span>}
        <hr />
      </div>
    ),
  },
}
```

### Custom Styling

Use the `classNames.separator` property to apply custom CSS classes:

```tsx
const theme = {
  classNames: {
    separator: 'my-2 border-t border-gray-200',
  },
}
```

```tsx

  slots: {
    Separator: ({ node }) => {

        return (
          <div className="px-2 py-1.5 text-xs font-medium text-muted-foreground">

          </div>
        )
      }
      return <div className="my-1 border-t border-border" />
    },
  },
}
```

## Search Behavior

Separators are always included in the flattened list but are not searchable. They don't match search queries and don't affect the search results of other items.

## Best Practices

1. **Use meaningful IDs**: Even though separators are visual only, give them descriptive IDs for debugging
2. **Don't overuse**: Too many separators can make menus feel cluttered
3. **Group related items**: Place separators between logically distinct groups of actions
4. **Consider alternatives**: For complex grouping, consider using Group nodes with headings instead

## Common Patterns

### Section Dividers

```tsx
{ kind: 'separator', id: 'divider-1' }
```

### Section Headers

```tsx
{ kind: 'separator', id: 'header-edit', label: 'Edit' }
```

### Dynamic Separators

```tsx
const nodes = [
  { kind: 'item', id: 'item1', label: 'Item 1' },
  ...(showExtraItems
    ? [
        { kind: 'separator', id: 'sep-extra' },
        { kind: 'item', id: 'extra', label: 'Extra Item' },
      ]
    : []),
]
```

## Notes

- Separators are not focusable and don't respond to keyboard navigation
- They are skipped when navigating with arrow keys
- Separators don't have selection state or callbacks
- They are automatically included when flattening menu structures
